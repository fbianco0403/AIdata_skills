### 2.1 ① tztek_t_personcost_bx（OA报销）→ TZTEK_T_COSTDETL

| COSTDETL字段 | 源字段 | 转换逻辑 |
|-------------|--------|----------|
| XMH | NULL | 初始为空,后续通过项目号匹配更新 |
| 年度 | LEFT(TJRQ,4) | 提交日期取年 |
| 账期 | LEFT(TJRQ,7) | 提交日期取年月 |
| 费用类别 | TZTEK_T_CostType_Account.CostType | 通过FINTYPE关联映射 |
| 科目 | FINTYPE | 直接取报销科目 |
| 流程提交日期 | TJRQ | 提交日期 |
| 财务审核日期 | FIDATE | 财务审核日期 |
| 预定人入职代码 | WORKCODE | 预订人工号 |
| 预订人 | LASTNAME | 预订人姓名 |
| 人员类别 | NULL | 空 |
| 所属中心 | NULL | 空 |
| 部门 | NULL | 空,后续通过TZTEK_T_BUSMODULE补全 |
| 项目号 | A.PJNO | 报销项目号 |
| 项目名称 | PJNAME | 报销项目名称 |
| 订单号 | LCBH | OA流程编号 |
| 结算金额 | ACTUALAMOUNT | 实际报销金额 |
| 费用用途 | '报销明细日期：'+ACTUALDATE+'，用途：'+NOTICE+','+USEFOR | 拼接描述 |
| 所属业务中心 | SYBMC | 事业部名称 |
| 事业部 | SSYWMK | 所属业务模块 |
| 分类 | '报销' | 固定值 |

**过滤条件：**
- LEFT(TJRQ,4) = 当年
- FIDATE非空（已审核）
- FINTYPE非空（有科目）
- SYBMC非空（有事业部）

## 三、存储过程内部处理逻辑字段级血缘

### 3.1 部门补全逻辑（TZTEK_T_BUSMODULE）

```
TZTEK_T_COSTDETL.部门 为空时:
  → 按预定人入职代码 + 流程提交日期年月 关联 TZTEK_T_BUSMODULE
  → 取每月最后一次记录(TIME最大)的SSBM
  → 仍为空 → 取该入职代码在TZTEK_T_BUSMODULE中最近一次记录
```

| TZTEK_T_BUSMODULE字段 | 用途 |
|----------------------|------|
| RZDM | 入职代码(关联键) |
| TIME | 日期(关联键,取每月最后一条) |

| SSYWMK | 所属业务模块 |
| SSYWZX | 所属业务中心 |

### 3.2 项目号匹配逻辑（TZTEK_V_LXXMHXX_ALL）

```
Step1: XMH为空 → 置为'public'
Step2: 项目号非空且非'public' → 通过 项目号 like XMH+'%' 关联 TZTEK_V_LXXMHXX_ALL
       → 更新 XMH, 事业部=XMDYBMMC, 所属业务中心=SYBMC
       (矽行事业部跳过此步骤)
```

| TZTEK_V_LXXMHXX_ALL字段 | 用途 |
|------------------------|------|

### 3.3 事业部名称规范化

```
大客户事业部 → 大客户其它
半导体事业部 → 半导体其它
光电事业部   → 光电其它
```

### 3.4 矽行系数打折

```
TZTEK_T_KEYWORD (KEYWORD_NAME='矽行系数', year(CRE_DT)=当年) → KEYWPRD_NUM

TZTEK_T_COSTDETL: 事业部='矽行' AND 分类 NOT IN ('其它_EBS付款','EBS_行政采购申请','报废_资产处置报废')
  → 结算金额 = 结算金额 × KEYWPRD_NUM

TZTEK_T_COSTDETL_CashFlow: 业务模块='矽行'
  → 结算金额 = cast(结算金额 as float) × KEYWPRD_NUM
```
