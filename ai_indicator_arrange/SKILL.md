# AI Indicator Arrange (指标数据治理编排)

## 概述
本技能是指标数据治理的总编排技能，当用户提供存储过程名称或指标名称时，自动调用各子技能完成完整的数据治理文档生成。

## 触发条件
- 用户提供存储过程名称，要求梳理数据治理相关内容
- 用户提供指标名称，要求生成完整的数据治理文档
- 用户要求对某个指标/存储过程进行全面分析

## 前置条件
- 已配置数据库连接信息（使用 db_connect_sqlserver 技能连接）
- Python 环境已安装 openpyxl、pyodbc 库

## 完整工作流程

### 第1步：血缘分析与算法 (ai_lineage_analysis)
1. 连接数据库，提取存储过程定义
2. 递归提取所有依赖的子存储过程、视图、函数
3. 追溯所有源表（包括跨库表）
4. 生成字段级血缘关系文档
5. 同步生成血缘思维导图（Mermaid格式，含思维导图/数据流向图/数据来源构成/分摊层级结构4种视图）
6. 输出：`{指标名}血缘分析与指标算法.md` 和 `{指标名}血缘思维导图.md`

### 第2步：逻辑算法说明 (ai_indicator_algorithm)
1. 分析存储过程的核心计算逻辑
2. 以开发人员和业务人员都能理解的方式描述算法
3. 不使用代码块，用自然语言+适当举例说明
4. 举例篇幅不宜过大
5. 输出：`{指标名}逻辑算法说明.md`

### 第3步：数据字典 (ai_data_dictionary)
1. 提取血缘中涉及的所有表的完整字段信息
2. 包含跨库表（通过 SELECT TOP 1 获取跨库表结构）
3. 每个字段必须补充中文名（根据数据内容+字段说明+代码注释）
4. 所有英文字母使用小写
5. 包含未在计算中使用但属于表结构的字段
6. 输出：`{指标名}数据字典.xlsx` 和 `{指标名}数据字典.md`

### 第4步：数据质量检验规则 (ai_data_quality_rules)
1. 基于数据字典生成质量检验规则
2. 规则类型包括：完整性校验、唯一性校验、值域校验、格式校验、业务规则校验、一致性校验
3. 每条规则提供可直接执行的SQL验证语句
4. 输出：`{指标名}数据质量检验规则.md`

### 第5步：数据清洗逻辑 (ai_data_cleaning_logic)
1. 基于血缘分析和数据字典生成清洗逻辑
2. 包含空值处理、格式标准化、异常值处理、重复数据处理、关联校验
3. 每条清洗规则提供可直接执行的SQL脚本
4. 输出：`{指标名}数据清洗逻辑.md`

### 第6步：元数据标准 (ai_metadata_standard)
1. 基于数据字典生成元数据标准
2. 包含：元数据分类标准、命名规范、数据类型标准、关系标准、质量标准、安全标准、变更管理、版本管理
3. 输出：`{指标名}元数据标准.xlsx`

### 第7步：数据标准 (ai_data_standard)
1. 基于数据字典生成业务级数据标准
2. 业务术语标准：覆盖数据字典中所有字段，同义词合并（如xmh/pjno/项目编号→项目号只制定一条标准）
3. 英文术语只用小写字母和下划线连接，不出现空格和其他符号
4. 包含：业务术语标准、参考数据标准、主数据标准、指标数据标准、数据认责标准、标准符合性规则、变更记录
5. 输出：`{指标名}数据标准.xlsx`

### 第8步：数据标准字段映射 (ai_standard_field_mapping)
1. 将数据标准中的术语映射到数据字典中的每个字段
2. 通过同义词映射确保所有字段都能匹配到标准术语
3. 输出：`{指标名}数据标准字段映射.xlsx`

### 第9步：业务数据地图 (ai_business_data_map)
1. 按照标准业务数据地图模板格式生成
2. 43列，5大区域：业务数据流程、数据安全、数据质量、数据标准、业务元数据
3. 业务域使用标准分类：项目域/研发域/销售域/生产域/采购域/仓库域/质量域/人事域/财务域/综合管理域/市场域
4. 子业务域使用标准分类（见 ai_business_data_map 技能详细列表）
5. 来源表单=表英文名，数据元来源=系统编码.表名
6. 所有英文字母全部小写
7. 输出：`{指标名}业务数据地图.xlsx`

## 调用方式

### 全量调用
当用户要求全面梳理时，按上述1-9步顺序依次调用所有子技能。

### 分步调用
当用户只需要某几项内容时，单独调用对应子技能：
- 只需血缘分析 → 调用 ai_lineage_analysis
- 只需算法说明 → 调用 ai_indicator_algorithm
- 只需数据字典 → 调用 ai_data_dictionary
- 只需质量规则 → 调用 ai_data_quality_rules
- 只需清洗逻辑 → 调用 ai_data_cleaning_logic
- 只需元数据标准 → 调用 ai_metadata_standard
- 只需数据标准 → 调用 ai_data_standard
- 只需标准映射 → 调用 ai_standard_field_mapping
- 只需数据地图 → 调用 ai_business_data_map

## 依赖关系
- 第3-9步依赖第1步的血缘分析结果（表清单）
- 第4-9步依赖第3步的数据字典
- 第7步依赖第3步的数据字典
- 第8步依赖第3步和第7步
- 第9步依赖第3步、第7步、第8步
- 因此建议按顺序执行，或确保前置依赖已完成
