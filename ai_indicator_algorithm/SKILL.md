# AI Indicator Algorithm (逻辑算法说明)

## 概述
对指定指标/存储过程的计算逻辑进行清晰描述，使开发部门和业务部门人员都能理解。不使用代码块，用自然语言描述，可适当举例。

## 触发条件
- 用户要求生成指标的算法说明
- 用户需要了解指标的计算逻辑
- 用户要求以业务可理解的方式描述算法

## 前置条件
- 已完成血缘分析（ai_lineage_analysis），掌握存储过程的完整逻辑
- 已连接数据库

## 执行步骤

### 1. 分析存储过程核心逻辑
- 识别存储过程的主要计算步骤
- 理解数据流转和转换规则
- 识别关键业务规则和计算公式

### 2. 提取核心指标定义
- 明确每个指标的业务含义
- 确定指标的计算公式和口径
- 识别指标的维度和粒度

### 3. 撰写算法说明
**核心原则**：
- **不用代码块**：用自然语言描述，不直接粘贴SQL/代码
- **统一叙述**：不分"开发视角"和"业务视角"分别叙述，而是写一份双方都能理解的说明
- **适当举例**：用具体数据举例说明计算过程，但举例篇幅不宜过大
- **准确描述**：确保算法描述与实际代码逻辑一致

### 4. 文档结构
1. **指标概述**：指标的业务定义和用途
2. **数据来源**：指标依赖的源数据说明
3. **计算逻辑**：
   - 数据采集与预处理步骤
   - 核心计算公式和规则
   - 分摊/汇总逻辑
   - 特殊处理规则（如剔除、调整）
4. **计算示例**：用1-2个具体场景举例说明
5. **注意事项**：边界条件、数据依赖、更新频率等

## 输出格式
文件名：`{指标名}逻辑算法说明.md`

## 注意事项
- 算法描述必须与实际存储过程代码一致，不能臆断
- 分摊逻辑要说明分摊基数和分摊比例的计算方式
- 多层级分摊要说明各层级的分摊对象和分摊方法
- 公共费用分摊要说明"公共"的定义和判定规则
- 金额计算要说明是否含税、币种转换规则
