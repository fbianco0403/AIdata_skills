# AI Standard Field Mapping (数据标准字段映射)

## 概述
将数据标准中的术语映射到数据字典中的每一个字段，建立标准与字段的对应关系。

## 触发条件
- 用户要求生成数据标准字段映射
- 用户需要查看每个字段对应的数据标准
- 用户要求建立标准与字段的追溯关系

## 前置条件
- 已完成数据字典（ai_data_dictionary）
- 已完成数据标准（ai_data_standard）

## 执行步骤

### 1. 加载数据字典和数据标准
- 读取数据字典Excel
- 读取数据标准Excel中的业务术语标准Sheet
- 读取同义词映射表 synonym_map.json

### 2. 字段到标准的映射逻辑
对数据字典中的每个字段：
1. 用字段中文名查 synonym_map，获取标准名
2. 用字段英文名查 synonym_map，获取标准名
3. 如果中文名和英文名映射到不同标准名：
   - 优先使用中文名映射的标准名
   - 如果中文名映射的标准名不在标准表中，使用英文名映射的标准名
4. 用标准名在数据标准中查找对应的术语信息

### 3. 生成映射Excel
左侧为数据字典原始信息，右侧为映射的数据标准信息。

## Excel列定义（20列）

**左侧：数据字典信息（A-K列）**
| 列名 | 说明 |
|------|------|
| 表分类 | 核心输出表/中间层源表/跨库源表/配置映射维度表 |
| 表中文名 | 表的中文描述 |
| 表英文名 | 表英文名（小写） |
| 字段英文名 | 字段英文名（小写） |
| 字段中文名 | 字段中文描述 |
| 字段说明 | 字段详细说明 |
| 数据类型 | 字段数据类型 |
| 长度 | 字段最大长度 |
| 是否可空 | YES/NO |
| 是否在计算中使用 | 是/否 |
| 表数据来源 | 数据来源说明 |

**右侧：数据标准信息（L-T列）**
| 列名 | 说明 |
|------|------|
| 标准术语编码 | BT-xxx格式 |
| 标准术语名称 | 标准中文名 |
| 标准术语英文名 | 小写+下划线 |
| 标准业务定义 | 术语的业务定义 |
| 标准业务规则/约束 | 术语的规则约束 |
| 标准同义词/别名 | 术语的所有同义词 |
| 标准提出部门 | 提出部门 |
| 标准认责部门 | 认责部门 |
| 标准状态 | 有效/无效 |

## 输出格式
文件名：`{指标名}数据标准字段映射.xlsx`

## 质量校验
- 映射率必须达到100%，所有字段都必须映射到标准术语
- 如有未映射字段，需检查 synonym_map 是否遗漏
- 同一标准术语对应的多个字段，其映射的标准信息应完全一致
