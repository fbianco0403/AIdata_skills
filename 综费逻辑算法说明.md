# 综费整体逻辑算法说明

---

## 一、综费指标定义

**综费（综合费用）** 是公司运营过程中产生的、不能直接归入产品制造成本的各类间接费用。它反映各事业部、各项目在运营中发生的"非生产性支出"。

**核心公式：**

> **项目综费 = 项目直接归属费用 + 公共费用分摊**

---

## 二、综费包含与排除

### 2.1 包含的费用

| 费用大类 | 包含内容 | 举例 |
|---------|---------|------|
| 差旅交通 | 国内机票、国际出差、住宿、打车、火车 | 员工出差机票、酒店住宿费 |
| 员工活动 | 餐费、餐补、餐厅消费、包厢招待 | 公司食堂餐补、部门聚餐 |
| 其它费用 | 服务费、维修费、模具费、NRE费用、投标保证金 | 第三方检测认证、设备维修 |
| 运输费用 | 物流费用、出口报关、进口报关 | 产品出口海运费 |
| 行政办公 | 水电煤、办公用品 | 车间电费分摊 |
| 招聘费用 | 招聘相关支出 | 猎头费 |
| 办事处费用 | 现场租房、办事处运营 | 驻外办事处租金 |
| 物料损耗 | 报废、拆机损失、辅料 | MES报废物料 |

### 2.2 排除的费用

| 排除项 | 原因 |
|--------|------|
| 资产采购 | 属于固定资产投资 |
| 物料采购 | 属于生产成本 |
| 投资并购 | 属于资本运作 |
| 基建费用 | 属于固定资产投资 |
| 借款/押金 | 属于资金往来 |
| 政府补贴/分红 | 属于收入/分配 |
| 增值税 | 税费 |
| 委外加工费 | 属于生产成本 |

## 三、综费计算流程

综费的计算分为五个步骤：**费用归集 → 维度补全 → 矽行系数 → 排除过滤 → 公共分摊**

### 3.1 第一步：费用归集

从6个外部系统采集费用数据，统一汇入费用明细表（TZTEK_T_COSTDETL），每笔费用记录包含：账期、费用类别、科目、项目号、事业部、业务模块、结算金额等维度。

| 数据来源 | 费用类型 | 关键过滤条件 |
|---------|---------|-------------|
| OA报销 | 报销费用 | 必须已审核（审核日期非空）、有科目、有事业部 |
| OA付款 | 付款费用 | 排除餐费/投资并购/投标保证金/退款/补贴/分红/基建/借款/押金；排除资产采购和物料采购大类 |
| 阿里商旅 | 差旅交通 | 机票取出行人优先、住宿取入住人优先、打车取预订人 |
| EBS应付 | 采购/其它费用 | 排除行政采购订单号（XZFY/XZGZ/XZWZ/SCFY开头）；科目通过映射表转换 |
| 行政采购 | 行政费用/物资 | 费用取含税金额，物资取未税金额 |
| 物料损耗 | 报废/拆机/辅料 | 取未税金额 |
| MueTec Odoo | 德国子公司费用 | 金额×固定汇率7.5×项目分摊率；排除2025年大类1240 |
| HRM餐补 | 餐费 | 本月实际餐补×工时占比 |
| 餐厅消费 | 餐费 | 餐厅消费+OA行政用餐 |
| 投标保证金 | 保证金 | 支付为正、归还为负；2026年起统计 |
| 水电煤 | 行政办公 | 总金额×车间面积占比×个人工时占比 |
| 运输费用 | 运输费用 | 排除技术服务费；通过序列号匹配项目号 |
| 年度调整 | 各类 | 线下录入的一次性调整 |

### 3.2 第二步：维度补全

费用归集后，部分记录缺少部门、项目号、事业部等信息，需要通过映射表补全。

**部门补全：** 按费用申请人的入职代码，在人员组织映射表（TZTEK_T_BUSMODULE）中查找。优先匹配同月份的记录，若未找到则取该人员最近一次的部门信息。

**项目号匹配：** 若项目号为空则标记为"public"（公共费用）。若项目号非空，则通过项目主数据（TZTEK_V_LXXMHXX_ALL）匹配，更新标准项目号、事业部、业务模块。矽行事业部跳过此步骤。

**事业部名称规范化：** 事业部名称统一为"XX其它"格式。例如"光电事业部"→"光电其它"，"智能事业部"→"智能其它"。"其它"→"运营中心"。

### 3.3 第三步：矽行系数打折

矽行是公司的半导体子公司，只按持股比例计入公司综费，因此其费用需要打折。

**系数来源：** 参数配置表（TZTEK_T_KEYWORD），KEYWORD_NAME='矽行系数'，当前值约0.1~0.171。

**打折范围：** 事业部="矽行"的费用，排除EBS付款、行政采购、资产处置报废三类（这三类由辉辉单独处理系数）。

**举例：** 矽行某笔差旅费原金额1000元，矽行系数0.171，则计入综费金额 = 1000 × 0.171 = 171元。

### 3.4 第四步：排除过滤

在维度补全和矽行系数处理之后，还需要执行以下排除：

| 排除规则 | 说明 |
|---------|------|
| 综费剔除清单 | 付款单号在TZTEK_T_COSTDETL_update中标记为"从综费剔除"的记录 |
| 增值税 | 科目为增值税的费用 |
| 委外加工费 | 特定内部交易付款单（FKSQ20231009045、FKSQ20240802011） |
| 特殊付款单 | 机器人Etas付款（FK25022100001），走无形资产 |

### 3.5 第五步：公共费用分摊

无法直接归属到具体项目的费用（项目号为空、"public"或"技术平台"，或事业部含"其它"），需要分摊到有订单额的项目上。

**分摊分两步识别：**

首先将费用分为"人员综费"和"其它综费"两类：
- **人员综费：** 费用类别属于综费类别（招聘费用、办事处费用、员工活动、行政办公、差旅交通）
- **其它综费：** 费用类别不属于综费类别（其它费用、运输费用、物料损耗等）

**分摊采用三级递进机制：**

| 级别 | 分摊对象 | 分摊方式 | 举例 |
|------|---------|---------|------|
| Level 1 业务模块级 | 同一业务模块内的公共费用 | 由该模块下有订单额的项目均摊 | 光电一部的公共差旅10万，光电一部有5个有订单项目 → 每个项目分摊2万 |
| Level 2 事业部级 | 事业部"其它"的公共费用 + Level1未匹配到的 | 由该事业部下所有项目均摊 | 光电"其它"的公共费用20万，光电事业部有20个项目 → 每个项目分摊1万 |
| Level 3 全公司级 | 运营中心/总裁办的公共费用 + Level2未匹配到的 | 由全公司所有项目均摊 | 运营中心公共费用50万，全公司有100个项目 → 每个项目分摊0.5万 |

**最终项目综费汇总：**

> **项目综费 = 项目直接归属费用 + 业务模块分摊金额 + 事业部其它分摊金额 + 其它级别分摊金额**

其中"结算金额汇总"= 直接费用 + Level1 + Level2（不含Level3，Level3单独展示）。

## 四、特殊场景算法

### 4.1 MueTec德国子公司费用

MueTec费用从Odoo系统获取，按以下方式计入综费：

> **MueTec项目费用 = Odoo原始金额(EUR) × 固定汇率7.5 × 项目费用分摊率**

项目费用分摊率来自配置表TZTEK_T_MueTecPJCostRate（cost_type='综费'），表示该项目在MueTec总费用中的占比。

**应发未实发补足：** 若某月MueTec数据尚未从Odoo同步，则取最近一次实发月份数据补足至当月，确保综费数据的连续性。

**举例：** MueTec 3月总费用10000EUR，项目A的分摊率为0.3，则项目A计入综费 = 10000 × 7.5 × 0.3 = 22500元。

### 4.2 水电煤分摊

水电煤费用按"车间面积 × 个人工时"两步分摊：

> **每行金额 = 水电煤总金额 × 车间实际面积占比 × 个人每月工时占比**

**举例：** 某月水电煤总金额10万，A车间面积占比40%，张三在A车间的工时占比5%，则张三分摊金额 = 10万 × 40% × 5% = 2000元。

### 4.3 投标保证金

投标保证金是"暂付性"费用，支付时计入综费，归还时冲减综费：

- **支付时：** 计入正数（增加综费）
- **归还时：** 计入负数（冲减综费）

**计入日期判定：** 归还的冲减计入哪个账期，取决于归还日期与预计归还日期的关系：
- 实际归还日期早于或等于预计归还日期+1天 → 计入实际归还日期所在账期
- 实际归还日期晚于预计归还日期+1天 → 计入预计归还日期+1天所在账期
- 尚未归还 → 计入预计归还日期+1天所在账期

**跨年冲减：** 若保证金在以前年度支付、今年归还，则只冲减今年净归还部分（今年归还金额 - 历年已归还金额），且冲减金额不超过原始支付金额。

**举例：** 2025年支付保证金100万，2026年归还100万，则2026年冲减100万（负数），综费净影响为0。

### 4.4 特别成本

特别成本是综费中需要单独标识的特殊费用，用于区分项目直接相关和间接相关的支出。

**包含三类：**

| 类型 | 说明 | 判定规则 |
|------|------|---------|
| 服务费 | 商务部门为项目采购的外部技术服务 | 商务部门 + 有项目号 → 是；商务部门 + 无项目号 → 数据修复；其它 → 否 |
| 视频动画 | 非中心部门为项目采购的视频制作 | 非中心 + 有项目号 → 是；非中心 + 无项目号 → 数据修复；中心所属 → 否 |
| NRE费用 | 一次性工程费用（存货编码=B01000007） | 直接标记为"是" |

**去重规则：** 业务支持合同审批流程中的服务费，会同时在付款申请中出现。为避免重复计算，付款申请中已关联业务支持合同的服务费需删除。

### 4.5 C19H457项目拆分（仅2024年）

C19H457是计量事业部的一个大项目，其费用需要按比例拆分到各子项目，拆分比例来自配置表TZTEK_T_ORDER_JiLiangSyb_XMH。

### 4.6 餐费

餐费数据来自三个渠道，统一归入"员工活动→餐费"：

| 渠道 | 说明 |
|------|------|
| HRM餐补 | 本月实际餐补标准 × 工时占比，按人员关联到项目 |
| 餐厅消费 | 食堂消费记录（水果饮料、包厢招待等），项目号标记为public |
| OA行政用餐 | 通过OA流程申请的用餐，可关联到具体项目号 |

## 五、输出表与用途

| 输出表 | 粒度 | 用途 |
|--------|------|------|
| TZTEK_T_COSTDETL | 每笔费用一行 | 综费最细粒度数据，支持多维度分析 |
| TZTEK_T_OACOST | 账期 + 事业部 | 运营指标总页面展示，按事业部汇总的费用支出 |
| TZTEK_T_COST | 项目 + 费用类型 | 项目维度综费汇总（含分摊），用于项目利润分析 |
| TZTEK_T_COSTDETL_CashFlow | 每笔费用一行 | 现金流视角费用，范围比综费更广（含餐费、保证金、资产采购） |
| TZTEK_T_COSTDETL_Specialcosts | 每笔费用一行 | 特别成本明细，标识服务费/视频动画/NRE |
| tztek_t_PoDetails | 每笔采购一行 | 累计采购合同额，含降价比分析 |

### 综费 vs 现金流的区别

现金流是综费的"扩大版"，额外包含了餐费、投标保证金、押金、资产采购、物料采购等综费不包含的项目。两者的核心区别在于统计口径：综费关注运营效率，现金流关注资金实际流出。

## 六、计算执行顺序

存储过程 TZTEK_P_C_OACOST 按以下顺序执行：

1. **费用归集**：依次插入餐厅消费→OA报销→OA付款→阿里商旅→EBS应付→物料损耗→行政费用→行政物资→MueTec→HRM餐补→投标保证金→水电煤→运输费用→年度调整
2. **维度补全**：补全部门→匹配项目号→规范化事业部名称
3. **矽行系数**：对矽行费用打折
4. **排除过滤**：剔除清单→增值税→委外加工费→特殊付款单
5. **汇总输出**：事业部级汇总→项目级分摊→现金流→特别成本→累计采购
6. **跨库同步**：将结果同步到DataDTRD库
