# AI Data Quality Rules (数据质量检验规则)

## 概述
基于数据字典生成可直接执行的数据质量检验规则，每条规则提供SQL验证语句。

## 触发条件
- 用户要求生成数据质量检验规则
- 用户需要对数据进行质量校验
- 用户要求制定数据质量标准

## 前置条件
- 已完成数据字典（ai_data_dictionary）
- 已连接数据库

## 执行步骤

### 1. 分析数据特征
基于数据字典中的字段信息，分析每类字段的特征：
- 主键字段：完整性、唯一性
- 外键字段：参照完整性
- 金额字段：数值范围、精度
- 日期字段：格式、逻辑一致性
- 编码字段：格式规范、值域范围
- 分类字段：值域范围

### 2. 生成质量规则
按以下6大类型生成规则：

#### 完整性校验
- 主键非空检查
- 必填字段非空检查
- 关联字段非空检查

#### 唯一性校验
- 主键唯一性检查
- 业务唯一键检查

#### 值域校验
- 枚举字段值域检查（如费用类别、事业部）
- 金额字段范围检查（>=0）
- 日期字段范围检查

#### 格式校验
- 项目号格式（字母+数字组合或public）
- 入职代码格式（SZ+6位数字）
- 日期格式（yyyy-mm-dd）
- 账期格式（yyyy-mm）

#### 业务规则校验
- 金额计算一致性（如：结算金额=含税金额-税额）
- 分摊金额合计校验
- 关联数据存在性校验

#### 一致性校验
- 跨表字段一致性
- 编码与名称对应一致性

### 3. 编写SQL验证语句
每条规则提供可直接执行的SQL：
```sql
-- 规则：项目号非空检查
SELECT COUNT(*) AS violation_count
FROM table_name
WHERE project_number IS NULL OR project_number = ''
```

## 输出格式
文件名：`{指标名}数据质量检验规则.md`

### 文档结构
1. 质量规则总览（规则数量统计）
2. 完整性校验规则
3. 唯一性校验规则
4. 值域校验规则
5. 格式校验规则
6. 业务规则校验
7. 一致性校验规则
8. 质量评分方案

每条规则包含：
- 规则编号
- 规则名称
- 规则描述
- 检查对象（表.字段）
- SQL验证语句
- 严重程度（高/中/低）

## 注意事项
- SQL语句必须可直接执行，表名和字段名使用实际值
- 规则要覆盖所有核心输出表和关键源表
- 金额类字段必须包含非负检查
- 外键字段必须包含参照完整性检查
