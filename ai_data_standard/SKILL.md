# AI Data Standard (数据标准)

## 概述
基于数据字典生成业务级数据标准Excel文件。核心要求：覆盖数据字典中所有字段，同义词合并为一条标准，英文术语只用小写+下划线。

## 触发条件
- 用户要求生成数据标准
- 用户需要数据治理中的数据标准（非元数据标准）
- 用户要求制定业务术语标准

## 前置条件
- 已完成数据字典（ai_data_dictionary）

## 执行步骤

### 1. 提取所有字段并识别同义词
从数据字典中提取所有字段，识别同义词：
- xmh / pjno / 项目编号 / attribute1 / project_number → 统一为"项目号"
- sybmc / 事业部名称 / attribute2 → 统一为"事业部"
- fylb / costtype → 统一为"费用类别"
- ssywmk / 所属业务模块 → 统一为"所属业务模块"

同义词识别方法：
- 字段英文名缩写与中文含义对照
- 不同表中含义相同的字段
- 代码中的别名映射

### 2. 构建同义词映射表
创建 synonym_map.json：
```json
{
  "项目编号": "项目号",
  "xmh": "项目号",
  "pjno": "项目号",
  "sybmc": "事业部",
  "事业部名称": "事业部"
}
```

### 3. 生成术语定义
为每个标准术语创建定义 term_defs.json：
```json
{
  "项目号": ["project_number", "项目的唯一标识编号", "格式：字母+数字组合", "项目管理部", "IT部", "备注"]
}
```

### 4. 英文术语规范
- **只用小写字母和下划线连接**
- **不出现空格和其他符号**
- 中文→英文翻译规则：
  - 项目号 → project_number
  - 事业部 → business_division
  - 费用类别 → expense_category
  - 结算金额 → settlement_amount

### 5. 生成Excel（7个Sheet）

#### Sheet1: 业务术语标准
覆盖数据字典中所有字段，同义词合并后每条标准包含：
- 术语编码（BT-001格式）
- 术语名称（标准中文名）
- 术语英文名（小写+下划线）
- 业务定义
- 业务规则/约束
- 同义词/别名（列出所有合并的同义词）
- 适用范围
- 提出部门
- 认责部门
- 状态
- 备注

#### Sheet2: 参考数据标准
枚举值类型的字段，列出代码值和含义：
- 费用类别代码表
- 事业部代码表
- 产品线代码表
- 币种代码表
- 公司代码表
- 项目属性代码表
- 项目状态代码表
- 是否标识代码表
等

#### Sheet3: 主数据标准
核心业务实体标准：
- 项目号、入职代码、组织架构、供应商编码、物料编码等

#### Sheet4: 指标数据标准
核心指标定义：
- 指标编码、名称、英文名、计算公式、口径说明、维度、粒度、更新频率

#### Sheet5: 数据认责标准
数据域认责：
- 数据域、认责部门、数据管理员、质量负责人

#### Sheet6: 标准符合性规则
标准校验规则：
- 参考数据校验、数据格式校验、业务规则校验、完整性校验、命名规范校验

#### Sheet7: 变更记录
标准变更历史

## 输出格式
文件名：`{指标名}数据标准.xlsx`

## 关键规则
1. **同义词必须合并**：xmh/pjno/项目编号只制定一条"项目号"标准
2. **英文术语规范**：只用小写+下划线，如 project_number
3. **全覆盖**：数据字典中每个字段都必须能映射到一条标准术语
4. **与元数据标准区分**：元数据标准管"元数据怎么管"，数据标准管"业务数据长什么样"
